# 자기 진화 프로토콜 (R1~R7)

> 정의 파일: `data/predict-evolution-protocol.json`

매 턴(30분) 분석 시 반드시 준수하는 7개 규칙.

## R1. 루틴 금지 (No Routine)

이전 턴과 실질적 차이가 없는 리포트를 생성하지 않는다.

- **체크:** 이전 리포트와 diff했을 때 숫자 조정만 있으면 '루틴 경고' 발동
- **조치:** 심화 전략(D1~D12) 중 미사용 1개+ 강제 실행
- **교훈:** 9~17세대에서 루틴 반복 → 18세대부터 프로토콜 도입으로 해결

## R2. 능동적 데이터 수집 (Active Collection)

데이터를 수동적으로 기다리지 않고 직접 수집한다.

- WebSearch로 최신 뉴스 탐색
- 기상·교통 등 실시간 데이터 확인
- SNS 트렌드 확인

## R3. 데이터 없이도 심화 (Deepen Without Data)

새 데이터가 없어도 기존 데이터로 더 깊이 분석할 수 있다.

- 기존 정보의 교차 검증·모순 탐색
- 가정(assumption) 명시 후 시뮬레이션
- 판례/통계 기반 정밀 추정
- 국제 유사사고 심층 비교

## R4. 전환점 감지 (Qualitative Shift Detection)

상황의 질적 전환점에서 프레임 자체를 재구성한다.

- **트리거:** 사망자 변동, 단계 전환, 정부 발표, 법적 조치, 국제 이슈화
- **조치:** 기존 프레임 유효성 재평가 → 새 국면에 맞는 분석 축 재설계

## R5. 정직한 자기 평가 (Honest Self-Assessment)

매 턴 이전 예측의 정확도를 정직하게 평가한다.

- **측정:** 방향 정확도, 시간 정확도, 규모 정확도, 시나리오 적중률
- **원칙:** 틀린 것을 인정하고 원인을 분석. 맞은 것도 왜 맞았는지 검증.

## R6. 상세 리포트 (Detailed Report)

결론뿐 아니라 분석 과정을 상세히 서술한다.

**6단계 서술 구조:**
1. 데이터 수집 과정 (소스, 키워드, 시각)
2. 분석 추론 과정 (해석, 가정, 교차 검증)
3. 시나리오 도출 과정 (확률 산정 근거)
4. 불확실성 식별 (확실 vs 추정)
5. 최종 결론
6. 자기 검증

**기준:** 심화 전략 500자+, 전체 2000자+

## R7. 추론 과정 기록 (Reasoning Trace)

리포트 하단에 `reasoningTrace` 섹션을 필수 포함한다.

**7단계 추론 구조:**
1. 초기 가설 (2~3개)
2. 가설 검증 (지지/반박 증거)
3. 가설 수정
4. 대안 탐색 (채택 안 한 해석과 이유)
5. 판단 근거
6. 인지 편향 점검 (확증/고정/가용성 편향)
7. 남은 의문

**스타일:** 1인칭 추론 서술 ("~라고 생각했으나", "~를 보고 수정했다")
**기준:** 300자+

## 안티패턴 (절대 하지 말 것)

- 숫자만 조정하고 '갱신했다'고 하지 않는다
- '정보 부족 모드'를 분석 회피 핑계로 쓰지 않는다
- 이전 세대 텍스트를 복사+붙여넣기 하지 않는다
- '변동 없음'으로 턴을 소비하지 않는다
- 결론만 쓰고 과정을 생략하지 않는다
- 확률·수치를 근거 없이 제시하지 않는다
- 추론 과정을 생략하지 않는다
