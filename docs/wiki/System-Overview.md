# 시스템 개요

## 온톨로지 기반 AI 예측이란

특정 상황(재난, 이벤트 등)을 **5개 도메인 온톨로지**로 구조화하고, **자기 진화 프로토콜**에 따라 매 턴(30분) 분석을 심화하는 예측 시스템.

## 핵심 특성

| 특성 | 설명 |
|------|------|
| **구조화 분석** | 5개 도메인(사건/공간/자원/인명/2차피해)으로 빠짐없이 분석 |
| **자기 진화** | 매 턴 이전 예측을 검증하고 새 분석 차원을 추가 |
| **능동 수집** | WebSearch로 최신 데이터를 직접 탐색 |
| **정직한 자기 평가** | 틀린 예측을 인정하고 편향을 점검 |
| **정량 모델** | 지수 감쇠, 가우시안 플룸, BCR 등 수학적 모델 적용 |

## 아키텍처

```
[데이터 소스]          [온톨로지 프레임워크]        [출력]
news.json         →   D1 사건 흐름            →   predict/gen-N.json
summary.json      →   D2 공간 영향            →   predict-report.json
WebSearch         →   D3 투입 자원            →   Issue 댓글
                  →   D4 인명 피해            →   대시보드 KPI
                  →   D5 2차 피해             →
                       ↕ 도메인 간 관계
                  →   E1~E6 확장 도메인        →

[자기 진화 엔진]
predict-evolution-protocol.json
├── R1 루틴 금지
├── R2 능동 수집
├── R3 데이터 없이도 심화
├── R4 전환점 감지
├── R5 정직한 자기 평가
├── R6 상세 리포트
└── R7 추론 과정 기록
```

## 진화 7단계

| Phase | 세대 범위 | 특성 |
|-------|----------|------|
| 1. Analysis | 1~7 | 개별 관점 분석 쌓기 |
| 2. Synthesis | 27/12 | 전 세대 분석 통합 |
| 3. Integration | 28/13 | 알고리즘 동시 가동 |
| 4. Judgment | 29/14 | 교훈·GO/NO-GO 판단 |
| 5. Simulation | 30/15 | 미래 시뮬·전이점 정의 |
| 6. Live Tracking | 31~33/16~19 | 실시간 추적·검증 |
| 7. Meta & Closure | 34~35/20 | Capstone·메타·최종 |
